getUpdates

از این متد برای دریافت آپدیت‌ها با استفاده از long polling استفاده کنید. این متد آرایه‌ای از Update را باز می‌گرداند.

GET https://tapi.bale.ai/bot<token>/getUpdates

پارامترها

پارامتر نوع الزام توضیحات
offset Integer اختیاری شناسه اولین آپدیتی که باید برگردانده شود. باید یک واحد بیشتر از بیشترین مقدار شناسه‌های مربوط به Update های دریافتی قبلی باشد. به طور پیش‌فرض، ابتدا Update هایی بازگشت داده می شوند که اولین Update تایید نشده را دارند. به محض اینکه getUpdates با یک offset بیشتر از update_id خود فراخوانی شود، Update به صورت تایید شده در نظر گرفته خواهد شد. از مقدار offset منفی هم می‌توان استفاده کرد. از مقدار منفی offset می‌توان برای دریافت Update ها از انتهای صف Update ها استفاده کرد. تمام Update های قبلی فراموش خواهند شد.
limit Integer اختیاری حداکثر تعداد آپدیت‌هایی که باید بازگردانده شوند. مقادیر بین ۱ تا ۱۰۰ پذیرفته می‌شوند. پیش‌فرض ۱۰۰ است.
timeout Integer اختیاری مدت زمان (بر حسب ثانیه) برای انتظار در حالت Long Polling. اگر آپدیتی در این مدت دریافت نشود، پاسخ خالی بازگردانده می‌شود. استفاده از این پارامتر باعث کاهش تعداد درخواست‌ها و بهبود کارایی می‌شود.

مثال درخواست

GET https://tapi.bale.ai/bot123456789:abcdIuZmK5qNEm2A1BhUaAg7MPJv1O9KCcBQB2ro/getUpdates?offset=100&limit=50&timeout=30

نکته مهم

به منظور جلوگیری از Update های تکراری، مقدار offset را بعد از هر پاسخ سرور، مجدد حساب کنید.

برای این کار، باید بزرگترین update_id دریافتی را پیدا کرده و به عنوان offset برای درخواست بعدی استفاده کنید (معمولاً به اضافه ۱).

الگوی استفاده

// الگوی ساده برای دریافت آپدیت‌ها
let offset = 0;

async function getUpdates() {
  const response = await fetch(
    `https://tapi.bale.ai/bot${TOKEN}/getUpdates?offset=${offset}&timeout=30`
  );
  const data = await response.json();
  
  if (data.ok && data.result.length > 0) {
    // پردازش آپدیت‌ها
    for (const update of data.result) {
      processUpdate(update);
      // بروزرسانی offset
      offset = update.update_id + 1;
    }
  }
  
  // فراخوانی مجدد
  setTimeout(getUpdates, 100);
}

مرحله بعد

برای تنظیم وب‌هوک به جای long polling، بخش setWebhook را مطالعه کنید.
