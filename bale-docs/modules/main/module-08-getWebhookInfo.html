getWebhookInfo

از این متد برای دریافت وضعیت فعلی وب‌هوک استفاده کنید. این متد به هیچ پارامتری نیاز ندارد.

در صورت اجرای موفقیت آمیز، یک شی WebhookInfo برگردانده می‌شود. اگر بازو در حال استفاده از متد getUpdates است، خروجی یک شی خواهد بود که فیلد url آن خالی است.

GET https://tapi.bale.ai/bot<token>/getWebhookInfo

پارامترها

این متد هیچ پارامتری نمی‌پذیرد.

مثال درخواست

GET https://tapi.bale.ai/bot123456789:abcdIuZmK5qNEm2A1BhUaAg7MPJv1O9KCcBQB2ro/getWebhookInfo

مثال پاسخ با وب‌هوک فعال

{
  "ok": true,
  "result": {
    "url": "https://example.com/webhook",
    "has_custom_certificate": false,
    "pending_update_count": 5,
    "ip_address": "203.0.113.1",
    "last_error_date": 1699999999,
    "last_error_message": "",
    "max_connections": 40,
    "last_synchronization_error_date": null
  }
}

مثال پاسخ بدون وب‌هوک

{
  "ok": true,
  "result": {
    "url": "",
    "has_custom_certificate": false,
    "pending_update_count": 0
  }
}

WebhookInfo

این شیء شامل اطلاعات درباره وضعیت وب‌هوک فعلی است.

فیلد نوع شرح
url String URL وب‌هوک تنظیم شده. اگر وب‌هوکی تنظیم نشده باشد، رشته خالی است.
has_custom_certificate Boolean آیا گواهی SSL سفارشی استفاده شده است؟
pending_update_count Integer تعداد آپدیت‌های در صف انتظار برای تحویل
ip_address String آدرس IP آخرین درخواست‌دهنده به وب‌هوک
last_error_date Integer زمان آخرین خطا (timestamp)
last_error_message String پیام آخرین خطا
max_connections Integer حداکثر تعداد اتصال‌های همزمان

کاربردها

  • بررسی اینکه آیا وب‌هوک به درستی تنظیم شده است
  • مشاهده تعداد آپدیت‌های در صف انتظار
  • عیب‌یابی خطاهای وب‌هوک
  • بررسی آدرس IP درخواست‌دهنده
