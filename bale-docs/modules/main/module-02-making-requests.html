ایجاد درخواست

برای ارتباط با API بازوی بله، باید درخواست‌های HTTP را به سرور ارسال کنید. در این بخش روش‌های مختلف ایجاد درخواست را بررسی می‌کنیم.

آدرس پایه API

تمام درخواست‌ها باید به آدرس زیر ارسال شوند:

https://tapi.bale.ai/bot<TOKEN>/<METHOD>

روش‌های ارسال درخواست

۱. استفاده از GET

برای متدهایی که پارامترهای کمی دارند، می‌توانید از درخواست GET استفاده کنید:

https://tapi.bale.ai/bot1234567890:ABCdef/sendMessage?chat_id=123456&text=سلام

۲. استفاده از POST (توصیه شده)

برای اکثر درخواست‌ها، بهتر است از POST استفاده کنید:

curl -X POST https://tapi.bale.ai/bot1234567890:ABCdef/sendMessage \
  -d "chat_id=123456789" \
  -d "text=سلام!"

فرمت داده‌ها

داده‌ها باید به صورت application/x-www-form-urlencoded یا multipart/form-data (برای آپلود فایل) ارسال شوند.

نمونه JSON Response

{
  "ok": true,
  "result": {
    "message_id": 123,
    "from": {
      "id": 123456789,
      "is_bot": true,
      "first_name": "Bot Name",
      "username": "mybot"
    },
    "chat": {
      "id": 987654321,
      "first_name": "User",
      "type": "private"
    },
    "date": 1234567890,
    "text": "سلام!"
  }
}

پارامترهای مشترک

پارامتر نوع توضیحات
chat_id String یا Integer شناسه گفتگو هدف یا @username کانال
message_id Integer شناسه پیام (برای ویرایش یا حذف)
text String متن پیام (۱ تا ۴۰۹۶ کاراکتر)
reply_markup String (JSON) کیبورد سفارشی به صورت JSON serialized

مدیریت خطاها

در صورت بروز خطا، API یک پاسخ JSON با فیلد ok: false برمی‌گرداند:

{
  "ok": false,
  "error_code": 400,
  "description": "Bad Request: chat not found"
}

کدهای خطای رایج

  • 400 Bad Request: پارامترهای نامعتبر
  • 401 Unauthorized: توکن اشتباه
  • 403 Forbidden: دسترسی غیرمجاز
  • 429 Too Many Requests: محدودیت نرخ

محدودیت نرخ (Rate Limiting)

API بله دارای محدودیت نرخ است. اگر بیش از حد مجاز درخواست ارسال کنید، خطای 429 دریافت خواهید کرد. در این صورت باید چند ثانیه صبر کنید و دوباره تلاش کنید.
