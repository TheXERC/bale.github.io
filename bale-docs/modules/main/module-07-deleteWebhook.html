deleteWebhook

در صورتی که تصمیم دارید مجدداً از متد getUpdates استفاده کنید، از این متد برای غیرفعال‌سازی اتصال وب‌هوک استفاده کنید.

در صورت موفقیت، مقدار True بازگردانده می‌شود.

POST https://tapi.bale.ai/bot<token>/deleteWebhook

پارامترها

این متد هیچ پارامتری نمی‌پذیرد.

مثال درخواست

POST https://tapi.bale.ai/bot123456789:abcdIuZmK5qNEm2A1BhUaAg7MPJv1O9KCcBQB2ro/deleteWebhook

مثال پاسخ موفق

{
  "ok": true,
  "result": true,
  "description": "Webhook was deleted"
}

کاربردها

  • تغییر از وب‌هوک به long polling (getUpdates)
  • غیرفعال کردن موقت دریافت آپدیت‌ها
  • عیب‌یابی و تست
  • انتقال بین محیط‌های مختلف (توسعه، تولید)

نکات مهم

  • پس از حذف وب‌هوک، آپدیت‌ها دیگر به URL قبلی ارسال نمی‌شوند
  • آپدیت‌های جدید در سرور بله ذخیره می‌شوند تا زمانی که با getUpdates دریافت شوند
  • می‌توانید بعداً با استفاده از setWebhook دوباره وب‌هوک را تنظیم کنید

جریان معمول

  1. تنظیم وب‌هوک با setWebhook
  2. دریافت آپدیت‌ها از طریق وب‌هوک
  3. در صورت نیاز به تغییر: deleteWebhook
  4. استفاده از getUpdates برای دریافت آپدیت‌ها

مرحله بعد

برای بررسی وضعیت وب‌هوک فعلی، بخش getWebhookInfo را مطالعه کنید.
