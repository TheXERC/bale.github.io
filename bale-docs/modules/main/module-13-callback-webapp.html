پرس‌وجوهای بازخورد و Mini App

ساختارهای داده‌ای مربوط به تعاملات کیبورد اینلاین و Mini App

CallbackQuery

این شیء نمایندهٔ یک پرس‌وجوی بازخورد از دکمه‌های اینلاین کیبورد است.

فیلد نوع توضیحات
id String شناسه یکتای پرس‌وجوی بازخورد
from User کاربری که دکمه را فشار داده است
message Message اختیاری. پیامی که دکمه اینلاین بخشی از آن است
inline_message_id String اختیاری. شناسه پیام اینلاین (اگر پیام در حالت inline ارسال شده باشد)
chat_instance String شناسه جهانی مربوط به چت و کاربر
data String اختیاری. داده‌های مرتبط با دکمه (callback_data)
game_short_name String اختیاری. نام کوتاه بازی (برای دکمه‌های بازی)

نکته: برای پاسخ به CallbackQuery، از متد answerCallbackQuery استفاده کنید.

WebAppData

این شیء نمایندهٔ داده‌های ارسالی از طرف Mini App است.

فیلد نوع توضیحات
data String داده‌های ارسالی از Mini App به صورت رشته
button_text String متن دکمه‌ای که برای ارسال داده استفاده شده است

مثال دریافت در Update:

{
  "update_id": 123456789,
  "message": {
    "message_id": 42,
    "from": {...},
    "chat": {...},
    "date": 1234567890,
    "web_app_data": {
      "data": "{\"action\":\"submit\",\"value\":\"test\"}",
      "button_text": "ارسال"
    }
  }
}

WebAppInfo

این شیء نمایندهٔ اطلاعات یک Mini App است.

فیلد نوع توضیحات
url String URL Mini App که باید باز شود

نکته: URL باید HTTPS باشد و دامنه معتبر داشته باشد.

CopyTextButton

این شیء نمایندهٔ یک دکمه کپی متن است.

فیلد نوع توضیحات
text String متنی که باید کپی شود

مثال در InlineKeyboardButton:

{
  "text": "کپی کد",
  "copy_text": {
    "text": "ABC123XYZ"
  }
}
