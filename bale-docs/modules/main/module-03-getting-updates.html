دریافت آپدیت‌ها

برای دریافت رویدادها و پیام‌های ارسالی به بازو، دو روش اصلی وجود دارد:

  1. Long Polling: استفاده از متد getUpdates
  2. Webhook: دریافت خودکار آپدیت‌ها از طریق webhook

مقایسه روش‌ها

ویژگی Long Polling Webhook
راه‌اندازی ساده نیاز به HTTPS
مناسب برای توسعه و تست محیط تولید
مصرف منابع بیشتر بهینه‌تر
تاخیر کم بسیار کم

شیء Update

هر آپدیت دریافتی یک شیء Update است که شامل اطلاعات رویداد رخ داده می‌باشد.

ساختار Update

فیلد نوع توضیحات
update_id Integer شناسه یکتای آپدیت (از صفر شروع می‌شود)
message Message پیام دریافتی (اختیاری)
callback_query CallbackQuery کوئری دکمه اینلاین (اختیاری)
edited_message Message پیام ویرایش شده (اختیاری)

متد getUpdates

این متد برای دریافت آپدیت‌ها از طریق Long Polling استفاده می‌شود.

پارامترها

پارامتر نوع الزامی توضیحات
offset Integer خیر شناسه آپدیت بعدی که باید دریافت شود. معمولاً برابر با update_id + 1 آخرین آپدیت پردازش شده
limit Integer خیر حداکثر تعداد آپدیت‌های قابل دریافت (پیش‌فرض: ۱۰۰)
timeout Integer خیر زمان انتظار به ثانیه برای Long Polling (پیش‌فرض: ۰)
allowed_updates Array of String خیر لیست انواع آپدیت‌های مجاز

مثال استفاده

curl "https://tapi.bale.ai/bot1234567890:ABCdef/getUpdates?offset=100&timeout=30"

پاسخ نمونه

{
  "ok": true,
  "result": [
    {
      "update_id": 100,
      "message": { ... }
    },
    {
      "update_id": 101,
      "message": { ... }
    }
  ]
}

نحوه پردازش آپدیت‌ها

  1. دریافت آپدیت‌ها با getUpdates
  2. پردازش هر آپدیت بر اساس نوع آن
  3. ذخیره update_id آخرین آپدیت پردازش شده
  4. ارسال offset = last_update_id + 1 در درخواست بعدی
